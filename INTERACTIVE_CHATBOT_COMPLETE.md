# 인터랙티브 대화형 노무 AI 챗봇 구현 완료

## 📋 개요

"진단 → 법적 분석 → 대안 제안 → 후속 질문" 구조의 단계별 대화형 노무 AI 챗봇이 완료되었습니다.

## 🎯 구현된 기능

### 1. 핵심 모듈

#### ConversationManager (`models/ConversationManager.js`)
- **세션 생성 및 관리**: 사용자별 대화 세션 추적
- **상태 관리**: 4단계 흐름 (diagnosis → analysis → solution → followup)
- **컨텍스트 유지**: 대화 내역, 수집된 정보, 법령/판례 저장
- **의도 분석**: 사용자 질문에서 카테고리, 긴급도, 감정 자동 감지
- **타임아웃 관리**: 30분 비활성화 시 세션 자동 삭제

#### DialogueTemplates (`models/dialogueTemplates.js`)
- **카테고리별 템플릿**: 임금, 해고징계, 근로시간, 휴가휴직, 기타
- **단계별 프롬프트**: 각 단계에 맞는 시스템 프롬프트 제공
- **진단 단계**: 초기 질문, 후속 질문, 필수 정보 필드 정의
- **분석 단계**: 법적 근거, 핵심 포인트, RAG 프롬프트 템플릿
- **해결 단계**: 단계별 해결 방법, 장단점, 긴급 조치 사항
- **후속 단계**: 추가 질문, 관련 주제 제안

#### DialogueFlowEngine (`models/DialogueFlowEngine.js`)
- **단계별 처리 로직**: 현재 단계에 따라 다른 응답 생성
- **정보 추출**: 정규식 패턴 매칭으로 사용자 입력에서 정보 추출
- **자동 단계 전환**: 충분한 정보 수집 시 다음 단계로 자동 이동
- **RAG 통합**: 각 단계에서 RAGAgent를 활용한 법령/판례 검색
- **공감 응답**: 사용자 의도(긴급, 부정적 감정)에 따른 맞춤 응답

#### ConversationStorage (`models/ConversationStorage.js`)
- **Firebase 통합**: Firestore를 통한 세션 및 메시지 영구 저장
- **세션 조회**: 사용자별, 세션별 대화 내역 조회
- **자동 정리**: 30일 이상 오래된 세션 자동 삭제
- **통계 기능**: 카테고리별 상담 분포 등 통계 제공

### 2. API 엔드포인트

#### 대화형 챗봇 API
```
POST   /api/chat/session/new          - 새 세션 생성
POST   /api/chat/message              - 메시지 전송
GET    /api/chat/session/:id          - 세션 조회
GET    /api/chat/session/:id/messages - 대화 내역 조회
GET    /api/chat/session/:id/summary  - 세션 요약
DELETE /api/chat/session/:id          - 세션 삭제
GET    /api/chat/sessions             - 활성 세션 목록
```

### 3. 웹 UI

#### chat.html
- **환영 화면**: 예시 질문 카드로 빠른 시작
- **단계 표시**: 현재 대화 단계를 시각적으로 표시
- **카테고리 표시**: 자동 감지된 노무 카테고리 표시
- **세션 정보**: 세션 ID 및 새 상담 버튼
- **반응형 디자인**: 모바일/데스크톱 모두 최적화

#### chat.js
- **자동 세션 생성**: 페이지 로드 시 자동으로 세션 시작
- **실시간 타이핑**: 입력 시 높이 자동 조절
- **타이핑 인디케이터**: AI 응답 대기 중 애니메이션
- **Enter 전송**: Enter로 전송, Shift+Enter로 줄바꿈
- **메시지 포맷팅**: 마크다운 스타일 간단 변환
- **예시 질문**: 클릭 한 번으로 예시 질문 전송

## 🔄 대화 흐름

### 1단계: 진단 (Diagnosis)
- **목적**: 사용자의 노무 문제 파악
- **동작**:
  1. 사용자 의도 자동 분석 (카테고리, 긴급도, 감정)
  2. 공감 응답 제공
  3. 카테고리별 초기 질문 제시
  4. 필수 정보 수집 (후속 질문)
  5. 충분한 정보 수집 시 2단계로 자동 전환

### 2단계: 법적 분석 (Analysis)
- **목적**: 법령과 판례에 기반한 법적 진단
- **동작**:
  1. 수집된 정보를 RAG 프롬프트에 삽입
  2. 노무 AI (RAGAgent)를 통한 법적 분석
  3. 관련 법령 조항 및 판례 제시
  4. 법적 쟁점 및 예상 결과 설명
  5. 추가 질문 가능, 완료 시 3단계로 전환

### 3단계: 대안 제안 (Solution)
- **목적**: 구체적인 해결 방법 안내
- **동작**:
  1. 단계별 해결 방법 제시 (1단계, 2단계, 3단계...)
  2. 각 방법의 소요 기간, 장단점 설명
  3. 상황별 권장 방법 제시
  4. 긴급 조치 사항 안내
  5. 특정 방법에 대한 상세 질문 가능

### 4단계: 후속 질문 (Followup)
- **목적**: 추가 정보 제공 및 관련 주제 안내
- **동작**:
  1. 후속 질문 목록 제시
  2. 관련 주제 제안
  3. 실행 지원 (서식, 절차 안내)
  4. 새 상담 시작 또는 종료

### 대화 예시

**사용자**: "임금을 3개월째 못 받고 있어요"

**AI (진단)**: "힘든 상황이시네요. 함께 해결 방법을 찾아보겠습니다.

임금 문제로 상담하시는군요. 구체적으로 어떤 문제가 있으신가요?

다음 중 해당하는 것을 선택하거나 자세히 설명해주세요:
1. 임금 체불 (월급을 못 받음)
2. 퇴직금 미지급
3. 연장·야간·휴일 수당 미지급
4. 임금 삭감
5. 기타 임금 문제"

**사용자**: "1번 임금 체불이요. 3개월 동안 300만원 못 받았어요"

**AI (진단 → 추가 질문)**: "네, 알겠습니다. 현재 재직 중이신가요, 아니면 퇴사하셨나요?"

**사용자**: "재직 중이에요"

**AI (진단 → 법적 분석 전환)**: "정보 감사합니다. 지금부터 법적으로 분석해드리겠습니다...

[법적 분석 내용]
1. 법적 진단: 근로기준법 제43조 위반, 악의적 임금 체불로 판단됩니다.
2. 관련 법령: 근로기준법 제43조 (임금 지급), 제109조 (벌칙 - 3년 이하 징역)
3. 청구 가능 금액: 원금 300만원 + 지연이자 약 60만원 (연 20%)
..."

(이후 대안 제안 → 후속 질문 단계로 계속 진행)

## 🎨 주요 특징

### 1. 맥락 유지 대화
- 이전 대화 내용을 기억하고 참조
- 단계별로 수집된 정보 누적
- 자연스러운 대화 흐름

### 2. 자동 카테고리 감지
- 10개 노무 카테고리 자동 분류
- 카테고리별 맞춤 템플릿 적용
- 정확도 높은 키워드 매칭

### 3. 공감적 응답
- 긴급한 상황 감지 시 우선 처리 안내
- 부정적 감정 감지 시 공감 표현
- 친절하고 이해하기 쉬운 말투

### 4. 법적 근거 제시
- 모든 답변에 법령 조항 명시
- 유사 판례 인용
- 출처 명확히 표시

### 5. 단계별 가이드
- 복잡한 해결 방법을 단계별로 분해
- 각 단계의 소요 시간 및 비용 안내
- 상황별 최적 방법 추천

7. **서식 자동 생성**: 내용증명, 진정서 등 자동 작성
8. **일정 관리**: 노동청 방문 일정, 소멸시효 알림 등
