## 1. 개요

문제 출제 스튜디오는 AI 기반의 문제 출제 도구입니다. 수학, 물리, 화학, 생명과학 등 다양한 과목의 문제를 효율적으로 작성하고 관리할 수 있습니다.

### 주요 기능
- 구조화된 문제 편집 (제시문, 보기/조건, 선지, 문제 지문)
- AI(LLM) 기반 문제 생성 및 변형
- 엔진 기반 문제 출제 규칙 적용
- 이미지/표 업로드 및 OCR
- 문제 미리보기 및 렌더링
- DB 저장 및 검수 워크플로우

## 2. 화면 구성

화면은 4개의 주요 패널로 구성되어 있습니다:

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│  📁 엔진    │  💻 엔진    │  🤖 LLM     │  📝 문제    │
│    폴더     │    코드     │    대화     │    편집     │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

### 패널 크기 조절
- 각 패널 헤더의 **◀ / ▶** 버튼으로 패널을 접거나 펼 수 있습니다
- LLM에게 "자료 패널 크기 키워줘", "레이아웃 초기화" 등의 명령으로 패널 제어 가능

## 3. 문제 편집 패널

문제 편집 패널은 가장 핵심적인 기능을 담당합니다.

### 3.1 상단 툴바

| 버튼 | 기능 |
|------|------|
| 📷 문제 입력 | 이미지로 문제 입력 |
| 📂 저장된 문제 불러오기 | DB에서 문제 로드 |
| 🔍 OCR | 이미지에서 텍스트 추출 |
| 🏷️ 라벨링 | 문제 분류 태그 지정 |
| 🔗 RAG 적용 | 문서 검색 기반 문제 생성 |
| 📋 검수 | 문제 검수 요청 |
| ⏳ 대기 저장 | 임시 저장 (검수 전) |
| ✅ 승인 저장 | 최종 승인 저장 |
| 🖨️ 렌더링 | 인쇄용 PDF 생성 |

### 3.2 편집 탭

- **✏️ 편집**: 문제 내용 편집
- **👁️ 미리보기**: 실제 출력 형태 미리보기
- **🖼️ 원본 이미지**: 업로드한 원본 이미지 확인

### 3.3 문제 편집 영역

#### 📄 제시문
문제의 배경 설명이나 상황을 입력합니다.

```
예: 다음은 이차함수 f(x)에 대한 설명이다.
```

#### 🖼️ 자료 (그림/표)
문제에 필요한 이미지나 표를 업로드합니다.

| 버튼 | 기능 |
|------|------|
| ➕ | 그림/표 박스 추가 (여러 개 가능) |
| 🗑️ | 업로드된 자료 삭제 |
| ⤢ | 패널 확대 |

**업로드 방법:**
- 영역 클릭하여 파일 선택
- 드래그 앤 드롭
- Ctrl+V로 클립보드 이미지 붙여넣기

#### 📋 보기/조건 박스
문제의 보기나 조건을 입력합니다.

| 버튼 | 기능 |
|------|------|
| ➕ | 보기/조건 박스 추가 |
| 🤖 | LLM 자동 타입 인식 |
| ⤢ | 패널 확대 |

**타입 선택:**
- **〈보기〉**: 보기 형식 (초록색 테두리로 표시)
- **〈조건〉**: 조건 형식 (주황색 테두리로 표시)
- **없음**: 박스 스타일 없이 표시

```
예시 (보기):
ㄱ. f(1) > 0
ㄴ. f'(2) = 0
ㄷ. f(3) < f(4)

예시 (조건):
(가) f(1) = 3
(나) f'(2) = 0
(다) f(x)의 최고차항의 계수는 1이다.
```

#### ⭐ 배점 & 🔢 선지 조건

**배점 설정:**
- 0.1점 ~ 20점까지 소수점 단위로 입력 가능
- 예: 3점, 4점, 3.2점 등

**난이도 선택:**
- 하 / 중 / 상 / 킬러

**선지 레이아웃:**
4가지 레이아웃 중 선택 가능

| 레이아웃 | 설명 | 예시 |
|----------|------|------|
| ①②③ | 한 줄에 하나씩 (기본) | 세로 배치 |
| ①②③ / ④⑤ | 3개 + 2개 | 첫 줄 3개, 둘째 줄 2개 |
| ①② / ③④⑤ | 2개 + 3개 | 첫 줄 2개, 둘째 줄 3개 |
| ①②③④⑤ | 한 줄에 모두 | 가로 배치 |

**선지 입력:**
각 선지(①~⑤)의 내용을 입력합니다.

#### ❓ 문제 지문
실제로 학생에게 묻는 질문을 입력합니다.

| 버튼 | 기능 |
|------|------|
| ➕ | 문제 지문 박스 추가 (연계 문제용) |
| ⤢ | 패널 확대 |

```
예: 위 조건을 만족시키는 f(x)에 대하여 f(5)의 값을 구하시오.
```

### 3.4 수식 입력

LaTeX 형식으로 수식을 입력할 수 있습니다.

```latex
\frac{1}{2}        → ½
\sqrt{x}           → √x
x^2 + y^2 = r^2    → x² + y² = r²
\int_0^1 x dx      → ∫₀¹ x dx
```

### 3.5 문제 네비게이션

- **문제 1/1**: 현재 문제 번호
- **+ 추가**: 새 문제 추가
- **◀ 이전 / 다음 ▶**: 문제 간 이동
- 숫자 버튼: 해당 문제로 바로 이동

### 3.6 레이아웃 관리

| 버튼 | 기능 |
|------|------|
| 🔄 초기화 | 패널 배치 초기화 |
| 💾 저장 | 현재 레이아웃 저장 |
| 🔓 잠금해제 | 패널 드래그 활성화 |

## 4. LLM 대화 패널

AI와 대화하며 문제를 생성하고 수정할 수 있습니다.

### 4.1 기본 사용법

메시지 입력창에 요청을 입력하고 ➤ 버튼 또는 Enter를 눌러 전송합니다.

### 4.2 문제 생성 요청 예시

```
"이 문제와 유사한 변형 문제 3개 만들어줘"
"난이도를 높여서 문제를 수정해줘"
"이차함수 관련 문제 출제해줘"
"선지의 오답을 더 그럴듯하게 수정해줘"
```

### 4.3 패널 제어 명령

```
"자료 패널 크기 키워줘"
"LLM 패널 숨겨줘"
"레이아웃 초기화"
"편집 패널만 크게 해줘"
```

### 4.4 엔진 선택

- **🤖 자동 선택**: 문제 내용에 따라 자동으로 적합한 엔진 선택
- 특정 엔진을 직접 선택하여 해당 규칙 적용 가능

### 4.5 관리 패널

**▼ 관리 패널** 버튼을 클릭하여 추가 설정에 접근할 수 있습니다.

## 5. 엔진 시스템

엔진은 문제 출제 규칙과 패턴을 정의한 파일입니다.

### 5.1 엔진 폴더 구조

```
📁 core (핵심 엔진)
├── 📄 math_problem.md      - 수학 문제 기본 규칙
├── 📄 graph_reasoning.md   - 그래프 추론 규칙
├── 📄 proof_logic.md       - 증명/논리 규칙
├── 📄 table_analysis.md    - 표 분석 규칙
└── 📄 science_experiment.md - 과학 실험 규칙

📁 plugin (플러그인 엔진)
├── 📁 diagram_2d          - 2D 다이어그램
├── 📁 geometry_2d         - 2D 기하학
├── 📁 graph_2d            - 2D 그래프
├── 📁 tree_graph          - 트리/그래프
├── 📁 table               - 표/행렬
└── ...
```

### 5.2 엔진 코드 편집

1. 왼쪽 **📁 엔진 폴더**에서 파일 선택
2. **💻 엔진 코드** 패널에서 내용 확인/수정
3. **💾 저장** 버튼으로 변경사항 저장

## 6. 문제 저장 및 관리

### 6.1 저장 워크플로우

```
문제 작성 → ⏳ 대기 저장 → 📋 검수 → ✅ 승인 저장
```

### 6.2 저장 유형

| 저장 유형 | 설명 |
|-----------|------|
| ⏳ 대기 저장 | 임시 저장 (검수 대기 상태) |
| ✅ 승인 저장 | 최종 승인 (문제 은행에 등록) |

### 6.3 불러오기

- **📂 DB에서 불러오기**: 저장된 문제 검색 및 로드
- **📂 저장된 문제 불러오기**: 최근 작업 문제 로드

## 7. 단축키 및 팁

### 7.1 단축키

| 단축키 | 기능 |
|--------|------|
| Ctrl+V | 클립보드 이미지 붙여넣기 (자료 영역) |
| Enter | LLM 메시지 전송 |

### 7.2 유용한 팁

1. **다중 박스 활용**: 그림이 2개 필요하거나 연계 문제인 경우 ➕ 버튼으로 박스 추가
2. **보기/조건 자동 인식**: 🤖 버튼을 클릭하면 내용을 분석하여 자동으로 타입 지정
3. **선지 레이아웃**: 선지 내용이 짧으면 가로 배치, 길면 세로 배치 권장
4. **미리보기 활용**: 편집 후 반드시 미리보기로 실제 출력 형태 확인
5. **엔진 선택**: 특정 유형의 문제는 해당 엔진을 선택하면 더 정확한 결과

### 7.3 문제 해결

| 문제 | 해결 방법 |
|------|-----------|
| 이미지가 업로드되지 않음 | 파일 크기 확인 (최대 10MB), 지원 형식 확인 (JPG, PNG) |
| 수식이 깨져 보임 | LaTeX 문법 확인, $...$ 또는 $$...$$ 사용 |
| 패널이 겹침 | 🔄 초기화 버튼 클릭 |
| LLM 응답이 없음 | 네트워크 연결 확인, 페이지 새로고침 |
