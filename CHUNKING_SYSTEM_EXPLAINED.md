# 현재 프로젝트의 RAG 청킹 시스템 설명

## 📊 현재 상황

✅ **법령 데이터만 업로드된 상태**
- `data/labor_laws/` 폴더에 법령 파일들 존재
- 판례(`data/labor_cases/`)는 아직 수집 중

✅ **청크 단위로 쪼개져서 RAG 검색됨**

## 🔧 청킹(Chunking) 작동 원리

### 1. 청킹이란?
법령 파일이 너무 크면 AI가 한 번에 처리할 수 없습니다. 그래서:
- **큰 문서를 작은 조각(청크)으로 분할**
- 각 청크를 벡터로 변환하여 저장
- 검색 시 관련 청크만 찾아서 응답 생성

### 2. 현재 시스템의 청킹 설정

#### 📜 법령 문서 청킹
```javascript
// models/laborSchemas.js
law: {
  whiteSpaceConfig: {
    maxTokensPerChunk: 512,      // 청크당 최대 512 토큰
    maxOverlapTokens: 64         // 청크 간 64 토큰 겹침
  }
}
```

**의미**:
- 법령 문서를 **512 토큰**(약 2048자) 단위로 분할
- 청크 사이에 **64 토큰**(약 256자) 겹침 → 문맥 유지
- **공백 기준(whiteSpace)** 분할 → 문장/단락 경계 존중

#### ⚖️ 판례 문서 청킹 (향후)
```javascript
case: {
  whiteSpaceConfig: {
    maxTokensPerChunk: 1024,     // 청크당 최대 1024 토큰
    maxOverlapTokens: 128        // 청크 간 128 토큰 겹침
  }
}
```

**의미**:
- 판례는 법령보다 **2배 큰 청크** (1024 토큰)
- 판시사항, 판결 이유 등이 길어서 더 큰 청크 필요
- 더 많은 겹침(128 토큰)으로 긴 논리 흐름 유지

## 🔄 청킹 및 RAG 검색 전체 프로세스

### 업로드 시 (법령 데이터 등록)

```
1. 법령 파일 준비
   └─> data/labor_laws/근로기준법.txt (예시)

2. uploadLaborLaw() 호출
   └─> RAGAgent.js (라인 655)

3. 청킹 설정 적용
   └─> LaborChunkingPresets.law
       { maxTokensPerChunk: 512, maxOverlapTokens: 64 }

4. uploadAndImportFile() 호출
   └─> RAGAgent.js (라인 155)
       ├─> Files API에 파일 업로드
       └─> File Search Store로 임포트

5. Google Gemini API 내부 처리
   ├─> 문서를 512 토큰씩 청크 분할
   ├─> 각 청크를 벡터 임베딩 변환
   ├─> File Search Store에 저장
   └─> 메타데이터 인덱싱 (법령명, 조항 등)
```

### 검색 시 (사용자 질문)

```
1. 사용자 질문
   └─> "근로기준법 제23조는 무엇인가요?"

2. askLabor() 호출
   └─> RAGAgent.js (라인 797)

3. 질문을 벡터로 변환

4. File Search Store에서 유사도 검색
   ├─> 모든 청크의 벡터와 비교
   ├─> 가장 관련 있는 청크 N개 찾기
   └─> (예: 근로기준법 제23조 포함 청크들)

5. 검색된 청크들을 컨텍스트로 전달

6. Gemini AI가 답변 생성
   └─> 찾은 청크들을 참고하여 답변 작성
```

## 🎯 청킹의 장점

### 1. **정확한 검색**
- 질문: "해고의 제한 조건은?"
- 검색: 제23조가 포함된 청크 3만 찾아옴
- 결과: 불필요한 정보 없이 정확한 답변

### 2. **문맥 유지 (오버랩)**
- 64 토큰 겹침으로 청크 경계에서도 문맥 연결
- 조항이 청크 사이에 걸쳐도 완전한 내용 제공

### 3. **효율적 처리**
- 전체 법령을 AI에 넣지 않고 관련 청크만
- 토큰 비용 절감 + 빠른 응답 속도

### 4. **확장 가능**
- 법령 추가 시 자동으로 청크 분할
- 수백 개 법령도 문제없이 처리

```javascript

const chunkingConfig = options.chunkingConfig || LaborChunkingPresets.law;
```
