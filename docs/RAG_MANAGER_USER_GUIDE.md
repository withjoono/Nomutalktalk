## 1. 시작하기

### RAG 관리자란?

RAG(Retrieval-Augmented Generation) 관리자는 AI 기반 문제 은행 시스템에서 **문서, 청크, 이미지 등의 데이터를 체계적으로 관리**하기 위한 도구입니다.

### 접속 방법

1. 메인 페이지에서 **"RAG 관리자"** 링크를 클릭합니다.
2. 또는 브라우저에서 직접 `/rag_manager.html` 경로로 접속합니다.

### 화면 구성

```
┌─────────────────────────────────────────────────────────┐
│  RAG 관리자 도구                                         │
│  ← 메인으로 돌아가기                                      │
├─────────────────────────────────────────────────────────┤
│  [통계 카드들: 총 문서, 총 청크, 미페어링 에셋 등]          │
├─────────────────────────────────────────────────────────┤
│  [탭 버튼들]                                             │
│  버전관리 | 에셋페어링 | 청크관리 | 자동청킹 | 임베딩 | ... │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [선택된 탭의 내용]                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 2. 대시보드 통계

화면 상단에 표시되는 6개의 통계 카드는 시스템의 현재 상태를 한눈에 보여줍니다.

| 통계 | 설명 | 색상 |
|------|------|------|
| **총 문서** | 시스템에 등록된 전체 문서 수 | 보라색 |
| **총 청크** | 생성된 모든 청크의 수 | 청록색 |
| **미페어링 에셋** | 아직 연결되지 않은 이미지/표의 수 | 핑크색 |
| **검토 대기** | 승인을 기다리는 항목 수 | 보라색 |
| **승인된 버전** | 승인 완료된 버전 수 | 청록색 |
| **초안 버전** | 아직 초안 상태인 버전 수 | 핑크색 |

> 💡 **팁**: 미페어링 에셋이 많으면 [에셋 페어링](#4-에셋-페어링) 탭에서 연결 작업을 진행하세요.

## 3. 버전 관리

문서나 청크의 변경 이력을 추적하고 관리합니다.

### 3.1 버전 히스토리 조회

특정 문서의 모든 버전 변경 내역을 확인합니다.

**사용 방법:**
1. **문서 ID** 입력란에 조회할 문서의 ID를 입력합니다.
2. **조회** 버튼을 클릭합니다.

**결과 테이블:**
| 열 | 설명 |
|---|---|
| 버전 | 버전 번호 (v1, v2, ...) |
| 상태 | draft(초안), pending_review(검토중), approved(승인됨), rejected(거부됨) |
| 생성일 | 버전이 생성된 날짜와 시간 |
| 변경 유형 | 어떤 종류의 변경인지 (수정, 추가 등) |
| 설명 | 변경 사항에 대한 설명 |
| 작업 | 초안 상태인 경우 승인 버튼 표시 |

### 3.2 버전 비교

두 버전 사이의 차이점을 비교합니다.

**사용 방법:**
1. **컬렉션** 드롭다운에서 비교할 대상을 선택합니다:
   - `청크`: 청크 버전 비교
   - `문서`: 문서 버전 비교
2. **버전 A ID**에 첫 번째 버전의 ID를 입력합니다.
3. **버전 B ID**에 두 번째 버전의 ID를 입력합니다.
4. **비교** 버튼을 클릭합니다.

**결과 화면:**
- 요약: 두 버전 간의 변경 요약
- 상세 비교: 각 필드별로 이전 값(빨간색)과 이후 값(초록색) 표시

## 4. 에셋 페어링

이미지, 표, 그래프 등의 에셋을 관련 문서나 청크와 연결합니다.

### 4.1 미페어링 에셋 확인

아직 연결되지 않은 에셋 목록을 확인합니다.

**사용 방법:**
1. **새로고침** 버튼을 클릭하여 최신 목록을 불러옵니다.

**각 에셋 카드에 표시되는 정보:**
- 에셋 유형 (image, table 등)
- 미리보기 이미지
- 캡션 또는 OCR 텍스트

**버튼 기능:**
| 버튼 | 기능 |
|------|------|
| **추천 받기** | AI가 가장 적합한 연결 대상을 자동으로 추천합니다 |
| **수동 페어링** | 직접 연결 대상을 지정합니다 |

### 4.2 페어링 생성

에셋을 특정 청크나 문서에 수동으로 연결합니다.

**입력 항목:**

| 필드 | 설명 | 예시 |
|------|------|------|
| **에셋 ID** | 연결할 에셋의 ID | `asset_abc123` |
| **대상 ID** | 연결할 청크 또는 문서의 ID | `chunk_xyz789` |
| **대상 유형** | 연결 대상의 유형 선택 | 문제, 교과서 본문, 해설, 설명 |
| **관계** | 에셋과 대상의 관계 | 아래 표 참조 |

**관계 유형:**

| 관계 | 설명 | 사용 예시 |
|------|------|----------|
| **illustrates (설명)** | 에셋이 대상 내용을 시각적으로 설명 | 개념 설명 그림 |
| **supplements (보충)** | 에셋이 대상 내용을 보충 | 추가 참고 자료 |
| **contains_data (데이터 포함)** | 에셋이 문제 풀이에 필요한 데이터를 포함 | 표, 그래프 |
| **shows_solution (풀이 표시)** | 에셋이 풀이 과정을 보여줌 | 풀이 그림 |
| **reference (참조)** | 일반적인 참조 관계 | 참고 이미지 |

## 5. 청크 관리

청크(문서의 작은 단위)를 조회하고 직접 생성합니다.

### 5.1 청크 유형별 조회

특정 유형의 청크만 필터링하여 조회합니다.

**청크 유형 버튼:**
- 📚 **교과서 본문**: 교과서의 텍스트 내용
- 🖼️ **교과서 이미지**: 교과서의 그림, 도표
- 📝 **문제 본문**: 문제 텍스트
- 🖼️ **문제 이미지**: 문제에 포함된 그림
- 📊 **문제 표**: 문제에 포함된 표
- ✏️ **해설 텍스트**: 정답 및 풀이 설명
- 🖼️ **해설 이미지**: 풀이 과정 그림

**조회 옵션:**
- ☑️ **승인된 것만**: 체크하면 승인된 청크만 표시

**결과 테이블 열:**
| 열 | 설명 |
|---|---|
| ID | 청크 고유 식별자 |
| 유형 | 청크 유형 |
| 상태 | draft, approved 등 |
| 내용 미리보기 | 처음 50자 표시 |
| 임베딩 | 임베딩 생성 여부 (🟢 완료 / 🟡 대기) |
| 작업 | 임베딩 생성 버튼 |

### 5.2 청크 직접 생성

새로운 청크를 수동으로 생성합니다.

**입력 항목:**

| 필드 | 설명 |
|------|------|
| **청크 유형** | 생성할 청크의 유형 선택 |
| **콘텐츠** | 청크의 실제 내용 입력 |
| **문서 유형** | 출처 문서의 유형 선택 |
| **자동 승인** | 체크하면 바로 승인 상태로 생성 |

**문서 유형 옵션:**
- 교과서
- 부교재
- 수능 기출
- 수능 모의
- 학교 시험
- 대학 논술
- 대학 면접

## 6. 자동 청킹

긴 텍스트를 자동으로 의미 있는 단위로 분할합니다.

### 6.1 청킹 전략

5가지 분할 전략을 제공합니다:

| 전략 | 아이콘 | 설명 | 권장 사용 상황 |
|------|--------|------|---------------|
| **문단 단위** | 📝 | 빈 줄을 기준으로 문단 분리 | 일반적인 교과서 본문 (권장) |
| **문장 단위** | 📖 | 문장 단위로 분리 후 묶음 | 짧은 문장이 많은 텍스트 |
| **의미 단위** | 🧠 | AI가 의미 기반으로 분할 | 복잡한 구조의 텍스트 |
| **고정 글자 수** | 📏 | 일정 글자 수로 분할 | 균일한 크기가 필요할 때 |
| **슬라이딩 윈도우** | 🪟 | 겹치는 부분을 두고 분할 | 연속성이 중요한 텍스트 |

### 6.2 전략별 옵션

#### 📝 문단 단위 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| 최소 문단 길이 | 50자 | 이보다 짧은 문단은 다음 문단과 합쳐짐 |

#### 📖 문장 단위 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| 청크당 문장 수 | 3개 | 하나의 청크에 포함될 문장 개수 |

#### 🧠 의미 단위 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| 최대 청크 수 | 10개 | AI가 생성할 최대 청크 개수 |

#### 📏 고정 글자 수 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| 최대 글자 수 | 500자 | 각 청크의 최대 길이 |
| 오버랩 | 50자 | 청크 간 중복되는 글자 수 |

#### 🪟 슬라이딩 윈도우 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| 윈도우 크기 | 500자 | 각 청크의 크기 |
| 스텝 크기 | 250자 | 윈도우 이동 간격 |

### 6.3 사용 방법

**Step 1: 텍스트 입력**
```
텍스트 입력란에 분할할 텍스트를 붙여넣습니다.
```

**Step 2: 옵션 설정**
1. 청킹 전략 선택
2. 전략별 세부 옵션 조정
3. 청크 유형 선택 (교과서 본문, 문제 본문, 해설 텍스트)
4. 문서 유형 선택
5. 필요시 자동 승인 / 자동 임베딩 체크

**Step 3: 미리보기 (권장)**
- **👁️ 미리보기** 버튼을 클릭하여 분할 결과를 먼저 확인합니다.
- 총 청크 수, 평균 길이 등 통계 정보 확인
- 각 청크의 내용 미리보기

**Step 4: 실행**
- 미리보기 결과가 만족스러우면 **✨ 청킹 실행** 버튼을 클릭합니다.
- 실행 완료 후 생성된 청크 ID 목록이 표시됩니다.

## 7. 임베딩 생성

청크에 대한 벡터 임베딩을 생성하여 유사도 검색을 가능하게 합니다.

### 7.1 임베딩이란?

임베딩은 텍스트를 숫자 벡터로 변환한 것입니다. 이를 통해:
- 유사한 내용의 청크를 빠르게 찾을 수 있습니다.
- AI가 관련 정보를 검색할 때 사용됩니다.

### 7.2 여러 청크 일괄 임베딩

**사용 방법:**
1. 청크 ID들을 쉼표(,)로 구분하여 입력합니다.
   ```
   chunk_abc123, chunk_def456, chunk_ghi789
   ```
2. **임베딩 생성** 버튼을 클릭합니다.

**결과:**
- 성공/실패 개수가 표시됩니다.

### 7.3 단일 청크 임베딩

**사용 방법:**
1. 하단의 입력란에 단일 청크 ID를 입력합니다.
2. **임베딩** 버튼을 클릭합니다.

> 💡 **팁**: [청크 관리](#5-청크-관리) 탭의 청크 목록에서 임베딩 버튼을 직접 클릭할 수도 있습니다.

## 8. 문제+자료 생성

문제와 관련 자료(이미지, 표)를 한 번에 생성하고 자동으로 연결합니다.

### 8.1 입력 항목

| 필드 | 필수 | 설명 |
|------|------|------|
| **문제 본문** | ✅ | 문제 텍스트 전체 |
| **정답** | ✅ | 정답 (예: ①, 42, 참) |
| **해설** | ❌ | 풀이 과정 설명 |
| **문서 유형** | ✅ | 문제 출처 유형 |
| **관련 이미지/표** | ❌ | 문제에 포함된 이미지 |
| **자동 승인** | ❌ | 생성 후 바로 승인 |

### 8.2 이미지 업로드

1. **클릭하거나 파일을 드래그하세요** 영역을 클릭합니다.
2. 이미지 파일을 선택합니다. (여러 개 선택 가능)
3. 업로드된 이미지는 미리보기로 표시됩니다.
4. 이미지 우측 상단의 ❌ 버튼으로 삭제할 수 있습니다.

### 8.3 생성 결과

생성 완료 후 다음 정보가 표시됩니다:
- 문제 청크 ID
- 해설 청크 ID (해설 입력시)
- 에셋 청크 개수
- 그룹 ID (연결된 항목들의 그룹 식별자)

## 9. 교과서+이미지 생성

교과서 본문과 관련 이미지를 한 번에 생성하고 자동으로 연결합니다.

### 9.1 입력 항목

| 필드 | 필수 | 설명 |
|------|------|------|
| **본문 텍스트** | ✅ | 교과서 본문 내용 |
| **문서 유형** | ✅ | 교과서 / 부교재 |
| **관련 이미지/표** | ❌ | 본문에 연결할 이미지 |
| **이미지 캡션** | ❌ | 각 이미지의 캡션 (쉼표로 구분) |
| **자동 승인** | ❌ | 생성 후 바로 승인 |

### 9.2 캡션 입력 방법

이미지 캡션은 쉼표(,)로 구분하여 입력합니다:
```
그림 1.1: 이차함수의 그래프, 표 1.2: 데이터 비교, 그림 1.3: 좌표평면
```

> ⚠️ **주의**: 캡션 개수와 이미지 개수가 다르면 순서대로 매칭되고, 부족한 경우 "그림 N" 형태로 자동 지정됩니다.

### 9.3 생성 결과

생성 완료 후 다음 정보가 표시됩니다:
- 본문 청크 ID
- 에셋 청크 개수
- 그룹 ID

## 10. 용어 설명

| 용어 | 설명 |
|------|------|
| **청크 (Chunk)** | 문서를 분할한 작은 단위. AI 검색과 처리에 최적화된 크기 |
| **에셋 (Asset)** | 이미지, 표, 그래프 등 멀티미디어 자료 |

| **버전 (Version)** | 문서나 청크의 변경 이력을 추적하기 위한 스냅샷 |
| **RAG** | Retrieval-Augmented Generation. 검색 기반 생성 AI 기술 |

## 11. 자주 묻는 질문

### Q: 청크 크기는 어느 정도가 적당한가요?
**A**: 일반적으로 200~500자가 권장됩니다. 너무 짧으면 맥락이 부족하고, 너무 길면 검색 정확도가 떨어집니다.

### Q: 자동 청킹에서 어떤 전략을 선택해야 하나요?
**A**:
- 교과서 본문 → **문단 단위** (권장)
- 긴 설명 텍스트 → **문장 단위** 또는 **슬라이딩 윈도우**
- 구조가 복잡한 텍스트 → **의미 단위**

### Q: 임베딩은 언제 생성해야 하나요?
**A**: 청크가 승인된 후 임베딩을 생성하는 것이 좋습니다. 자동 청킹 시 "자동 임베딩" 옵션을 체크하면 자동으로 생성됩니다.

### Q: 페어링은 왜 필요한가요?
**A**: 이미지나 표와 관련 텍스트를 연결하면:
- AI가 문맥을 더 잘 이해합니다.
- 사용자에게 관련 자료를 함께 보여줄 수 있습니다.
- 검색 결과의 품질이 향상됩니다.

### Q: 버전 관리의 장점은 무엇인가요?
**A**:
- 변경 이력 추적 가능
- 잘못된 수정 시 이전 버전으로 복원 가능
- 여러 사람이 검토 후 승인하는 워크플로우 지원

### Q: 에러가 발생하면 어떻게 하나요?
**A**:
1. 화면에 표시된 에러 메시지를 확인합니다.
2. 입력값이 올바른지 확인합니다.
3. 브라우저를 새로고침 후 다시 시도합니다.
4. 문제가 지속되면 관리자에게 문의하세요.

## 부록: 단축키 및 팁

### 작업 효율을 높이는 팁

1. **미리보기 활용**: 자동 청킹 전 항상 미리보기를 확인하세요.
2. **일괄 처리**: 여러 청크의 임베딩을 한 번에 생성하면 시간을 절약할 수 있습니다.
3. **자동 승인 주의**: 중요한 데이터는 자동 승인보다 수동 검토를 권장합니다.
4. **캡션 미리 작성**: 이미지 업로드 전 캡션을 미리 작성해 두면 편리합니다.

### 권장 워크플로우

```
1. 텍스트 준비
   ↓
2. 자동 청킹 (미리보기 → 실행)
   ↓
3. 청크 검토 및 승인
   ↓
4. 이미지가 있으면 에셋 페어링
   ↓
5. 임베딩 생성
   ↓
6. 완료!
```
