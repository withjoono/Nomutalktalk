# RAG vs Fine-tuning: 노무 AI 딥러닝 업그레이드 가이드

## 🎯 핵심 질문: "RAG에서 딥러닝으로 가면 뭐가 달라질까?"

## 📊 현재 시스템 (RAG)

### 작동 방식
```
사용자 질문
    ↓
벡터 검색 (법령 청크에서 관련 내용 찾기)
    ↓
검색된 청크 + 질문 → Gemini AI
    ↓
답변 생성
```

### 특징
- **모델은 그대로** (Gemini 기본 모델)
- **데이터는 외부에 저장** (File Search Store)
- **검색해서 찾은 정보로 답변**
- 모델이 노동법을 "학습"한 게 아님

---

## 🧠 딥러닝 (Fine-tuning)으로 업그레이드

### 작동 방식
```
1. 준비: 노동법 + 질문-답변 쌍 데이터셋 구축
    ↓
2. 학습: Gemini 모델을 노동법 데이터로 재학습
    ↓
3. 추론: 사용자 질문 → Fine-tuned 모델 → 답변
    (외부 검색 없이도 법령 지식이 모델 안에 내재화)
```

### 특징
- **모델 자체가 변함** (노동법 전문가 모델로 진화)
- **지식이 모델 내부에 각인**
- **검색 없이도 답변 가능** (더 빠름)
- 모델이 노동법을 "암기"하고 "이해"함

---

## 🔄 비교표

| 구분 | RAG (현재) | Fine-tuning (딥러닝) | 하이브리드 (RAG + Fine-tuning) |
|------|------------|----------------------|----------------------------------|
| **모델 지식** | ❌ 일반 지식만 | ✅ 노동법 전문 지식 내재화 | ✅ 전문 지식 + 최신 정보 |
| **답변 속도** | 🟡 중간 (검색 필요) | ✅ 빠름 (검색 불필요) | 🟡 중간 |
| **정확도** | 🟡 검색 품질에 의존 | ✅ 매우 높음 (학습된 지식) | ✅✅ 가장 높음 |
| **최신 정보** | ✅ 즉시 업데이트 (파일 추가) | ❌ 재학습 필요 | ✅ RAG로 보완 |
| **비용** | 🟢 저렴 | 🔴 고비용 (학습 비용) | 🟡 중간 |
| **구축 시간** | 🟢 빠름 (수일) | 🔴 오래 걸림 (수주~수개월) | 🟡 중간 |
| **데이터 요구** | 법령 파일만 있으면 됨 | 수천~수만 개 Q&A 필요 | 법령 + Q&A |
| **출처 명시** | ✅ 쉬움 (검색된 청크) | ❌ 어려움 (모델 내부 지식) | ✅ RAG 부분에서 가능 |
| **환각(Hallucination)** | 🟡 중간 (검색 결과 기반) | ⚠️ 높을 수 있음 (학습 부족 시) | 🟢 낮음 (RAG로 검증) |

## 🚀 Fine-tuning 시 변화

### 1. **답변 품질 향상**

#### RAG (현재)
```
질문: "정당한 이유 없는 해고란?"

검색: 근로기준법 제23조 청크 찾기
    ↓
답변: "근로기준법 제23조에 따르면, 사용자는 근로자를 정당한 이유 없이 
      해고할 수 없습니다. 정당한 이유란..."
      (검색된 내용 기반 답변)
```

#### Fine-tuned (딥러닝)
```
질문: "정당한 이유 없는 해고란?"

모델 내부 지식 활용
    ↓
답변: "정당한 이유 없는 해고는 근로기준법 제23조에서 금지하고 있으며,
      판례에서는 다음 4가지 요건을 모두 충족해야 정당한 해고로 인정합니다:
      1. 사회통념상 고용관계를 계속할 수 없을 정도
      2. 해고회피 노력
      3. 합리적 해고기준
      4. 공정한 해고대상자 선정
      (대법원 1999다10935 참조)
      
      실무에서는..."
      (더 깊은 이해와 맥락, 판례까지 자연스럽게 통합)
```

**차이점**:
- Fine-tuned 모델은 **법령 + 판례 + 실무 지식이 통합**되어 있음
- **더 자연스럽고 전문적인** 답변
- **맥락 이해가 깊어짐**

### 2. **복잡한 질문 처리 능력**

#### RAG (현재)
```
질문: "5인 미만 사업장에서 임금 체불과 부당해고가 동시에 발생했을 때, 
      구제 절차는 어떻게 달라지나요?"

문제점:
- 여러 청크를 조합해야 함 (임금 + 해고 + 5인 미만 예외)
- 법령 간 관계 파악이 어려움
- 답변이 산만해질 수 있음
```

#### Fine-tuned (딥러닝)
```
질문: "5인 미만 사업장에서 임금 체불과 부당해고가 동시에 발생했을 때,
      구제 절차는 어떻게 달라지나요?"

모델이 이미 알고 있음:
✅ 5인 미만 사업장은 부당해고 구제신청 불가 (근로기준법 적용 제외)
✅ 임금 체불은 5인 미만도 근로기준법 적용 (제43조)
✅ 따라서 임금은 노동청 진정, 해고는 민사소송만 가능

답변:
"5인 미만 사업장은 특별한 상황입니다.
1. 임금 체불: 근로기준법 제43조는 사업장 규모 무관하게 적용되므로
   노동청 진정 또는 민사소송 가능
2. 부당해고: 5인 미만은 근로기준법 제23조 적용 제외로
   노동위원회 구제신청 불가, 민사소송(해고무효 확인)만 가능
   
따라서 병행 전략은..."

(복잡한 법령 간 관계를 자연스럽게 통합 설명)
```

### 3. **추론 및 예측 능력**

#### RAG (현재)
- 검색된 내용만 답변 가능
- 새로운 상황에 대한 추론 제한적

#### Fine-tuned (딥러닝)
- **유사 사례 추론**: "이 사례는 대법원 2015다xxx와 유사하므로..."
- **결과 예측**: "이 경우 승소 가능성은 약 70%로 예상됩니다. 왜냐하면..."
- **전략 제안**: "현재 증거 상황을 보면, A 방법보다 B 방법이 유리합니다"

### 4. **대화 맥락 이해**

#### RAG (현재)
```
사용자: "임금 체불이 3개월이에요"
AI: "임금 체불에 대한 법령은..."

사용자: "그럼 소멸시효는?"
AI: [소멸시효 일반론 답변] ← 임금 체불 맥락 놓칠 수 있음
```

#### Fine-tuned (딥러닝)
```
사용자: "임금 체불이 3개월이에요"
AI: "3개월 임금 체불은... [상세 답변]"

사용자: "그럼 소멸시효는?"
AI: "임금채권의 소멸시효는 3년입니다. (근로기준법 제49조)
     3개월 체불은 아직 시효 내이므로 청구 가능합니다."
     ← 이전 대화(3개월 체불) 맥락 완벽히 이해
```

## 📈 실제 노무 AI에 Fine-tuning 적용 시나리오

### Phase 1: RAG 기반 (현재) ✅
- **목적**: 빠른 프로토타입, 개념 검증
- **데이터**: 법령 파일만
- **시간**: 1주
- **비용**: 거의 무료

### Phase 2: 데이터 수집 (3~6개월)
```
1. 사용자 질문 수집
   - 실제 상담 기록 3,000개 이상
   - "임금 체불이 3개월인데..." → "노동청 진정 절차는..."

2. 전문가 답변 작성
   - 노무사가 검증한 표준 답변
   - 근거 법령 + 판례 명시

3. 데이터셋 구축
   {
     "질문": "5인 미만 사업장 부당해고",
     "답변": "5인 미만 사업장은...",
     "근거": ["근로기준법 제23조", "대법원 2020다..."],
     "카테고리": "해고",
     "난이도": "중급"
   }
   
   최소 5,000개 이상 필요
```

### Phase 3: Fine-tuning 실행 (1~2개월)
```
1. Google Gemini Fine-tuning API 사용
   또는 OpenAI GPT-4 Fine-tuning

2. 학습 데이터 업로드
   - 5,000~10,000개 Q&A 쌍

3. 하이퍼파라미터 튜닝
   - Learning rate, Batch size 등

4. 평가 및 반복
   - 테스트 세트로 정확도 측정
   - 목표: 95% 이상 정확도
```

### Phase 4: 하이브리드 시스템 구축 (1개월)
```
사용자 질문
    ↓
1차: Fine-tuned 모델로 답변 생성
    ↓
2차: RAG로 최신 법령/판례 검색
    ↓
3차: 1차 + 2차 통합하여 최종 답변
    ↓
출처 명시 (RAG에서 가져온 청크)
```

## 🎯 언제 Fine-tuning을 해야 할까?

### Fine-tuning이 필요한 경우 ✅

1. **높은 정확도 요구**
   - 법률 자문처럼 실수가 허용되지 않는 분야
   - 95% 이상 정확도 필요

2. **복잡한 추론 필요**
   - 여러 법령을 조합한 분석
   - 판례 적용 및 결과 예측

3. **대량의 사용자 확보**
   - 월 10만 건 이상 질문 처리
   - Fine-tuning 비용 대비 효과적

4. **충분한 데이터 확보**
   - 5,000개 이상 고품질 Q&A
   - 전문가 검증 완료

5. **예산 확보**
   - 초기 투자 $10,000 이상 가능
   - 지속적 유지보수 예산

### RAG로 충분한 경우 ✅ (현재 권장)

1. **빠른 출시 필요**
   - 1~2주 내 서비스 시작

2. **자주 변하는 법령**
   - 법 개정이 잦아 즉시 업데이트 필요

3. **예산 제한**
   - 초기 투자 최소화

4. **데이터 부족**
   - 아직 충분한 Q&A 데이터 없음

5. **출처 명시 중요**
   - 답변 근거를 명확히 보여줘야 함

## 🏆 최적의 전략: 하이브리드 접근

### 단계별 로드맵

#### 1단계: RAG로 시작 (0~3개월) ✅ 현재
- 법령 데이터 업로드
- 기본 질의응답 시스템 구축
- **사용자 질문 수집 시작** ⭐

#### 2단계: 데이터 수집 (3~12개월)
- 실제 사용자 질문 3,000개 이상 수집
- 전문가(노무사) 답변 작성
- 카테고리별 데이터셋 구축

#### 3단계: Fine-tuning 1차 (12~15개월)
- 수집된 데이터로 첫 Fine-tuning
- 핵심 카테고리 먼저 (임금, 해고)

#### 4단계: 하이브리드 시스템 (15~18개월)
```python
def answer_question(question):
    # 1. Fine-tuned 모델로 초안 생성
    draft = fine_tuned_model.generate(question)
    
    # 2. RAG로 최신 법령 검색
    relevant_chunks = rag_search(question)
    
    # 3. 통합 답변 생성
    final_answer = integrate(draft, relevant_chunks)
    
    # 4. 출처 명시
    final_answer += format_sources(relevant_chunks)
    
    return final_answer
```

#### 5단계: 지속적 개선 (이후)
- 6개월마다 Fine-tuning 업데이트
- 새로운 판례 즉시 RAG에 추가
- 사용자 피드백 반영

## 🔬 기술적 구현 방안

### Google Gemini Fine-tuning (2024년 기준)

```javascript
// 1. 데이터셋 준비
const trainingData = [
  {
    "input": "5인 미만 사업장 부당해고 구제 방법은?",
    "output": "5인 미만 사업장은 근로기준법 제23조가 적용되지 않아 노동위원회 구제신청이 불가합니다. 대신 민사소송(해고무효 확인)을 통해서만 구제받을 수 있습니다. 절차는..."
  },
  // ... 5,000개 이상
];

// 2. Fine-tuning 실행
const tunedModel = await genAI.models.fineTune({
  baseModel: 'gemini-1.5-pro',
  trainingData: trainingData,
  hyperparameters: {
    learningRate: 0.001,
    batchSize: 16,
    epochs: 3
  }
});

// 3. 사용
const response = await tunedModel.generateContent(
  "임금 체불 3개월인데 어떻게 하나요?"
);
```

### 하이브리드 시스템 구현

```javascript
// RAGAgent 확장
class HybridLaborAI extends RAGAgent {
  constructor(apiKey, fineTunedModelName) {
    super(apiKey);
    this.fineTunedModel = fineTunedModelName;
  }

  async answerWithHybrid(question) {
    // 1. Fine-tuned 모델로 초안
    const draft = await this.generateWithFineTuned(question);
    
    // 2. RAG로 최신 정보 검색
    const ragResults = await this.askLabor(question);
    
    // 3. 통합
    const finalAnswer = await this.integrate(draft, ragResults);
    
    return {
      answer: finalAnswer,
      sources: this.extractSources(ragResults),
      confidence: this.calculateConfidence(draft, ragResults)
    };
  }
}
```

```
4. 18개월: 하이브리드 시스템 구축

최종 목표:
- Fine-tuned 모델: 전문 지식 내재화
- RAG: 최신 정보 즉시 반영
- 최고의 정확도 + 최신성
```
